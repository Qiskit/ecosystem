## Application

Helps run or work with higher-than-circuit or domain-specifc level of abstraction.

## Circuit simulator

Simulates a quantum computer on your local classical computer

## Compute provider

[Qiskit primitives](https://quantum.cloud.ibm.com/docs/en/api/qiskit/primitives) is a computational 
model introduced by the Qiskit for accessing quantum resources for executing of quantum programs.
Clients that provide access to remote service by exposing
Qiskit primitives interfaces belong to this category.
The service itself might execute the quantum program in a real quantum hardware or a circuit simulator.


## Converter

Converts Qiskit objects (such as QuantumCircuit) for use in other programs or tools

## Game

Games or playful educational resources

## Paper artifact

Code or dataset needed to reproduce a specific paper. Probably, without long-term support.

## Qiskit addon

Enables algorithm discovery at the utility scale

## SDK

An SDK, or software development kit, provides an interface to build software on top of it.
Sometimes, they are called "toolkits".
In the context of the Qiskit Ecosystem, an SDK provides ways to build and transform data 
structures such as quantum circuits.
The canonical example of a quantum SDK is [Qiskit SDK](https://www.ibm.com/quantum/qiskit).


## Tooling

Helps to develop quantum programs, such as IDEs, debuggers, or other development tooling.

## Transpiler plugin

A Qiskit transpiler plugin implements the plugin interface that allows to hook non-qiskit 
code into the Qiskit transpilation pipeline.
If a project is in this category, consider tagging it with one of these labels:
[`transpiler stage`](../labels#transpiler-stage),
[`unitary synthesis`](../labels#unitary-synthesis), and
[`high-level synthesis`](../labels#high-level-synthesis).

Tutorial:
[Create a Qiskit plugin](https://quantum.cloud.ibm.com/docs/en/guides/create-transpiler-plugin)


## Other

Projects that extend, interface with, or use Qiskit in interesting ways
